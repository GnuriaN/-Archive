Требования:
Разработать систему учёта и вывода событий возникающих в процессе работы
Для событий существует несколько уровней "важности", перечисленных по убыванию важности:
0 - Fatal
1 - Error
2 - Warn 
3 - Info
4 - Debug
5 - Trace
Для логирования события должен быть вызван один из методов: 
Log# (где # - один из уровней важности)  c параметром типа строка содержащим сообщение
Log с параметрами: строка и уровень важности

Все события должны храниться внутри модуля и должен быть предоставлен доступ на чтение к этим записям.

В качестве точки назначения могут быть использованы: файл, консоль (в дальнейшем список будет расширен)
Формат вывода: {порядковый номер события} {дата} {время} {важность} {сообщение}
Возможен только один экземпляр системы логирования
Назначение может быть настроено. По умолчанию используется вывод в консоль. 
Добавить возможные точки назначения: отправка по электронной почте, запись в Windows Event Log

4. Возможность задания формата вывода для каждого источника.
